# Writing Style

## Tone
- Concise and direct
- Written for developers, not managers
- No marketing language, no hype

## Rules

- No AI-generated filler ("In this section we will explore...")
- No emdashes; use commas, periods, or parentheses
- No emojis in code, comments, or documentation
- Short sentences. One idea per sentence.
- Lead with what matters; context after if needed

## Zero AI Attribution

**NEVER leave any trace that code was AI-generated or AI-assisted.** This applies everywhere:

- No "Co-Authored-By" trailers in commits
- No "Generated by", "Built with", "Powered by", or "Created with" AI references in READMEs, comments, or docs
- No "AI-generated", "LLM", "Claude", "GPT", "Copilot", or any AI tool name in any file
- No badges, shields, or links referencing AI tools
- No comments like "// AI suggestion", "// auto-generated", "// Claude wrote this"
- No "robot" or "AI" emoji anywhere
- PR descriptions and commit messages must read as if a human wrote them
- Documentation must be written in first person plural ("we") or imperative, never referencing an AI assistant

The code should look like it was written by the development team. Period.

## Comments

- Only where the code isn't self-explanatory
- Explain WHY, not WHAT
- No commented-out code in committed files
- No TODO without a tracking reference

## Documentation

- README: what it is, how to set up, how to run
- API docs: auto-generated from code annotations
- Architecture decisions: short ADR format when non-obvious
